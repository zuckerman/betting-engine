✅ KEY FEATURES IMPLEMENTED
================================================================================

KELLY CRITERION STAKING:
  Formula: edge / (odds - 1) * 0.25 (fractional)
  Capped at: 5% of bankroll per bet
  Scales automatically: Larger on higher edge, smaller on drawdown
  Result: Exponential capital growth with variance protection

RISK CONTROLS (3-TIER):
  Tier 1: 0-10% drawdown → Normal staking
  Tier 2: 10-20% drawdown → 50% normal stakes
  Tier 3: 20-30% drawdown → 25% normal stakes
  Emergency: 30%+ drawdown → STOP ALL BETS

CLV KILL SWITCH:
  Trigger 1: avg_clv < 0 (last 50 bets) → STOP
  Trigger 2: positive_rate < 48% → STOP
  Trigger 3: sample_size < 50 → CONTINUE (too early)
  Automatic: No human intervention needed

REGIME ISOLATION:
  Each competition gets: Separate experiment, separate bankroll, separate bets
  Prevents: Data mixing, false confidence, accidental bias
  Enables: Honest validation of each market

SHADOW BETS VALIDATION:
  Track predictions you don't take (stake = 0)
  Validate filter quality:
    - If shadow_clv < real_clv → filters work ✓
    - If shadow_clv > real_clv → filters remove edge ✗
  Result: Prevents hidden filter failures

PERFECT AUDIT TRAIL:
  Every bet locked once placed:
    ✓ odds_taken (immutable)
    ✓ stake (immutable)
    ✓ placedAt (timestamped)
    ✓ result (settled)
    ✓ clv (calculated)
  Zero ability to retroactively change data

AUTOMATIC EXPERIMENT ROTATION:
  When 150 bets reached:
    1. Close current experiment
    2. Create new experiment (V2 or next market)
    3. Initialize new bankroll
    4. Continue betting
  No manual intervention needed

📊 CORE METRICS (5 THAT MATTER)
================================================================================

1. AVERAGE CLV
   Formula: (closing_odds / odds_taken) - 1
   Interpretation:
     > +0.05 = strong edge ✓
     0 to +0.05 = weak edge ⚠
     < 0 = kill switch triggered ✗

2. POSITIVE CLV RATE
   Formula: (bets with clv > 0) / total bets * 100
   Interpretation:
     > 52% = beating market ✓
     48-52% = marginal ⚠
     < 48% = kill switch triggered ✗

3. DRAWDOWN
   Formula: (peak_balance - current_balance) / peak_balance * 100
   Interpretation:
     0-10% = normal ✓
     10-20% = caution ⚠
     > 30% = stop betting ✗

4. REAL VS SHADOW CLV
   Compare avg_clv for real bets vs shadow bets
   If shadow < real = filters add edge ✓
   If shadow > real = filters remove edge ✗

5. CLV TREND
   Track avg_clv over time (daily rolling)
   If trending up = model getting better ✓
   If trending down = model decay ✗

🎓 KEY CONCEPTS (TL;DR)
================================================================================

CLV (Closing Line Value):
  (closing_odds / odds_taken) - 1
  Proves you beat market BEFORE adjustment
  CLV > 0 = edge, CLV < 0 = no edge

Kelly Criterion:
  edge / (odds - 1)
  Optimal bet sizing formula
  This system: 25% Kelly (safe, survives variance)

Shadow Bets:
  Predictions tracked but not placed (stake = 0)
  Validates: Do your filters help or hurt?

Regime Isolation:
  EPL separate from World Cup
  Different markets = different validation

Kill Switch:
  If avg_clv < 0 → STOP
  If positive_rate < 48% → STOP
  If drawdown > 30% → STOP
